# 하이라이트 조회 문제 해결 체크리스트

## 🔍 현재 상황
- 앱에서 하이라이트 조회 시 빈 배열 반환
- DB에는 `mb_id = '2'`인 데이터 4건 존재
- 서버 코드는 이미 수정 완료

## ✅ 수정 완료 사항

### 1. 서버 코드 수정
- ✅ `mb_id`를 명시적으로 문자열로 변환
- ✅ 상세한 디버깅 로그 추가
- ✅ CAST를 사용한 대체 쿼리 로직 추가
- ✅ 데이터베이스 연결 정보 로그 출력
- ✅ 쿼리 실행 전후 상세 로그

### 2. 추가된 로그
- 데이터베이스 연결 정보 (DB_NAME 확인)
- 인증 과정 (user.id → mb_id 변환)
- 쿼리 실행 전 파라미터 확인
- 쿼리 결과 상세 정보
- 대체 쿼리 시도 결과

## 📋 다음 단계 (반드시 수행)

### 1. 서버 재빌드 및 재시작
```bash
cd /Volumes/DATA/000_Projects/webdev/www/pamp

# 빌드 캐시 삭제
rm -rf .next

# 재빌드
npm run build

# 서버 재시작 (사용 중인 프로세스 매니저에 따라)
pm2 restart pamp --update-env
# 또는
systemctl restart pamp
# 또는
npm run start
```

### 2. 서버 로그 확인
서버 콘솔에서 다음 로그를 확인하세요:

#### a) 서버 시작 시
```
[DB 연결 정보]
  - DB_HOST: 115.68.222.162
  - DB_USER: root
  - DB_NAME: pamp  ← 이 값이 pamp인지 확인!
  - NODE_ENV: production
```

**중요**: `DB_NAME`이 `pamp`가 아니라 `pamp_dev`라면 환경 변수를 수정해야 합니다.

#### b) 하이라이트 조회 요청 시
```
[하이라이트 인증] 회원 인증 성공:
  - user.id (원본): 2 타입: number
  - mb_id (변환): 2 타입: string

[하이라이트 조회] 인증 결과:
  - auth.mb_id: 2 타입: string
  - mbId (최종): 2 타입: string
  - is_member: true

[하이라이트 조회] 전체 데이터 통계:
  - mb_id별 데이터 개수: [{"mb_id":"2","cnt":4}]

[하이라이트 조회] 쿼리 실행:
  - 쿼리: SELECT * FROM tb_bible_highlight WHERE mb_id = ? ORDER BY created_at DESC LIMIT ?
  - 파라미터: ["2",5]
  - mbId 값: "2"
  - mbId 길이: 1
  - mbId 문자 코드: [50]

[하이라이트 조회] 쿼리 결과:
  - 조회된 행 수: 4  ← 이 값이 0이면 문제!
```

### 3. 환경 변수 확인
서버의 환경 변수가 올바른지 확인:

```bash

pm2 env pamp

# 또는 환경 변수 파일 확인
cat .env.production
# 또는
cat .env
```

확인해야 할 값:
- `DB_NAME=pamp` (프로덕션)
- `DB_HOST=115.68.222.162`
- `DB_USER=root`

### 4. 직접 쿼리 테스트
서버에서 직접 쿼리를 실행하여 데이터 존재 확인:

```sql
-- 1. 데이터베이스 선택
USE pamp;

-- 2. 전체 데이터 확인
SELECT * FROM tb_bible_highlight;

-- 3. mb_id별 데이터 개수
SELECT mb_id, COUNT(*) as cnt FROM tb_bible_highlight GROUP BY mb_id;

-- 4. mb_id = '2'인 데이터 확인
SELECT * FROM tb_bible_highlight WHERE mb_id = '2';

-- 5. CAST를 사용한 쿼리 (대체 방법)
SELECT * FROM tb_bible_highlight WHERE CAST(mb_id AS UNSIGNED) = 2;
```

## 🐛 문제 해결 가이드

### 문제 1: DB_NAME이 pamp_dev로 설정됨
**증상**: 서버 로그에서 `DB_NAME: pamp_dev`로 표시

**해결**:
1. 환경 변수 파일 수정: `DB_NAME=pamp`
2. 서버 재시작

### 문제 2: 서버가 재시작되지 않음
**증상**: 로그에 새로운 디버깅 정보가 표시되지 않음

**해결**:
1. 서버 프로세스 완전 종료
2. `.next` 폴더 삭제
3. `npm run build` 재실행
4. 서버 재시작

### 문제 3: 쿼리 결과가 0건
**증상**: 로그에 "조회된 행 수: 0" 표시

**확인 사항**:
1. `mbId 값`이 `"2"` (문자열)인지 확인
2. `전체 데이터 통계`에서 `mb_id: "2"`인 데이터가 있는지 확인
3. `대체 쿼리 시도 (CAST 사용)` 결과 확인

**해결**:
- CAST 쿼리로 데이터를 찾았다면, 서버 코드에서 CAST를 사용하도록 수정
- 또는 데이터베이스의 `mb_id` 값에 공백이나 특수 문자가 있는지 확인

### 문제 4: 서버 로그가 보이지 않음
**해결**:
1. 서버 로그 파일 위치 확인
2. PM2를 사용하는 경우: `pm2 logs pamp --lines 100`
3. systemd를 사용하는 경우: `journalctl -u pamp -n 100`
