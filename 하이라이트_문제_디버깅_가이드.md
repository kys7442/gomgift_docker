# 하이라이트 조회 문제 디버깅 가이드

## 🔍 현재 상황

- **앱**: 사용자 ID 2로 하이라이트 조회 요청
- **DB**: `mb_id = '2'`인 데이터 4건 존재 (직접 쿼리 확인)
- **서버 응답**: `success: true, data: []` (빈 배열)
- **서버 코드**: 이미 `String(user.id)`로 변환하도록 수정 완료

## 📋 확인 사항

### 1. 서버 재시작 확인

Next.js 프로덕션 모드에서는 코드 변경 후 반드시 재빌드 및 재시작이 필요합니다:

```bash
# 프로덕션 빌드
npm run build

# 프로덕션 서버 재시작
pm2 restart pamp
# 또는
systemctl restart pamp
# 또는 사용 중인 프로세스 매니저에 따라
```

### 2. 서버 로그 확인

서버 콘솔에서 다음 로그를 확인하세요:

```
[하이라이트 인증] 회원 인증 성공:
  - user.id (원본): 2 타입: number
  - mb_id (변환): 2 타입: string

[하이라이트 조회] 인증 결과:
  - auth.mb_id: 2 타입: string
  - mbId (최종): 2 타입: string
  - is_member: true

[하이라이트 조회] 쿼리 실행:
  - 쿼리: SELECT * FROM tb_bible_highlight WHERE mb_id = ? ORDER BY created_at DESC LIMIT ?
  - 파라미터: ['2', 5]
  - mbId 값: "2"
  - mbId 길이: 1
  - mbId 문자 코드: [50]

[하이라이트 조회] 쿼리 결과:
  - 조회된 행 수: 0
  - ⚠️ 데이터가 없습니다.
```

### 3. 데이터베이스 연결 확인

서버 시작 시 다음 로그가 표시됩니다:

```
[DB 연결 정보]
  - DB_HOST: 115.68.222.162
  - DB_USER: root
  - DB_NAME: pamp  ← 이게 pamp인지 pamp_dev인지 확인!
  - NODE_ENV: production
```

**중요**: `DB_NAME`이 `pamp`인지 확인하세요. `pamp_dev`라면 개발 DB를 참조하고 있는 것입니다.

### 4. 직접 쿼리 테스트

서버에서 직접 쿼리를 실행하여 데이터 존재를 확인:

```sql
-- 1. 데이터베이스 확인
USE pamp;

-- 2. 전체 데이터 확인
SELECT * FROM tb_bible_highlight;

-- 3. mb_id별 데이터 개수 확인
SELECT mb_id, COUNT(*) as cnt FROM tb_bible_highlight GROUP BY mb_id;

-- 4. mb_id = '2'인 데이터 확인
SELECT * FROM tb_bible_highlight WHERE mb_id = '2';

-- 5. mb_id 타입 확인
SELECT mb_id, TYPEOF(mb_id) FROM tb_bible_highlight LIMIT 1;
```

### 5. 환경 변수 확인

서버의 환경 변수가 올바르게 설정되어 있는지 확인:

```bash
# PM2를 사용하는 경우
pm2 env <app_name>

# systemd를 사용하는 경우
systemctl show-environment

# 또는 .env 파일 확인 (있는 경우)
cat .env.production
```

확인해야 할 환경 변수:
- `DB_HOST`: 데이터베이스 호스트
- `DB_USER`: 데이터베이스 사용자
- `DB_PASSWORD`: 데이터베이스 비밀번호
- `DB_NAME`: **반드시 `pamp`여야 함** (프로덕션)
- `JWT_SECRET`: JWT 시크릿 키

## 🐛 가능한 원인

### 원인 1: 데이터베이스 연결이 잘못됨
- `DB_NAME`이 `pamp_dev`로 설정되어 있음
- 해결: 환경 변수를 `pamp`로 변경하고 서버 재시작

### 원인 2: 서버가 재시작되지 않음
- 코드 변경 후 빌드/재시작을 하지 않음
- 해결: `npm run build` 후 서버 재시작

### 원인 3: 캐시 문제
- Next.js 빌드 캐시가 남아있음
- 해결: `.next` 폴더 삭제 후 재빌드

### 원인 4: mb_id에 숨겨진 문자
- mb_id에 공백이나 특수 문자가 포함되어 있을 수 있음
- 해결: 서버 로그에서 `mbId 문자 코드` 확인

### 원인 5: 데이터베이스 연결 풀 문제
- 이전 연결이 남아있어 다른 DB를 참조할 수 있음
- 해결: 서버 완전 재시작

## ✅ 해결 단계

1. **서버 로그 확인**
   ```bash
   # PM2 로그 확인
   pm2 logs pamp --lines 100
   
   # 또는 systemd 로그
   journalctl -u pamp -n 100
   ```

2. **환경 변수 확인 및 수정**
   ```bash
   # 환경 변수 파일 수정 (있는 경우)
   nano .env.production
   # DB_NAME=pamp 확인
   ```

3. **완전 재빌드 및 재시작**
   ```bash
   # 빌드 캐시 삭제
   rm -rf .next
   
   # 재빌드
   npm run build
   
   # 서버 재시작
   pm2 restart pamp --update-env
   ```

4. **서버 로그에서 확인할 내용**
   - `[DB 연결 정보]`에서 `DB_NAME`이 `pamp`인지 확인
   - `[하이라이트 인증]`에서 `mb_id`가 `"2"` (문자열)로 변환되는지 확인
   - `[하이라이트 조회] 쿼리 결과`에서 조회된 행 수 확인

5. **직접 쿼리로 검증**
   ```sql
   -- 서버가 실행하는 것과 동일한 쿼리
   SELECT * FROM pamp.tb_bible_highlight WHERE mb_id = '2' ORDER BY created_at DESC LIMIT 5;
   ```

## 📞 추가 디버깅

서버 로그에 위의 정보가 표시되지 않으면:

1. 서버가 실제로 재시작되었는지 확인
2. 코드 변경이 배포되었는지 확인
3. 서버 로그 레벨이 올바른지 확인
