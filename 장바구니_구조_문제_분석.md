# 장바구니 구조 문제 분석 보고서

## 문제 발견

**현상**: 장바구니에서 상품당 여러 옵션이 있어야 하는데 1개의 옵션만 표시됨

**원인**: `shop/cart.php`의 `get_cart_items_data()` 함수에서 AOV 상품 옵션 조회 시 특정 `ct_id`만 조회하고 있음

---

## 🔍 문제 상세 분석

### 1. 문제 위치

**파일**: `shop/cart.php`  
**함수**: `get_cart_items_data()`  
**라인**: 113줄

### 2. 문제 코드

```php
// ❌ 문제: 특정 ct_id만 조회
$aov_sql = " select a.ct_id, ...
           from {$g5['g5_shop_cart_table']} a ...
          where a.ct_id = '{$row['ct_id']}'  // ← 문제: 특정 ct_id만 조회
            and a.it_id = '{$row['it_id']}'
            and a.od_id = '$s_cart_id'
            ...
```

**문제점**:
- `group by a.it_id`로 상품별로 그룹화한 후
- 첫 번째 `ct_id`만 사용하여 해당 상품의 다른 옵션들을 조회하지 못함
- 결과적으로 같은 상품(it_id)에 여러 옵션이 있어도 1개만 표시됨

### 3. 올바른 로직 (참고: boxcart.skin.php)

**파일**: `theme/kiwi/skin/shop/basic/boxcart.skin.php`  
**라인**: 49-53줄

```php
// ✅ 올바른 로직: 같은 상품의 모든 옵션 조회
$sql2 = " select a.ct_option, a.ct_qty, a.ct_id, ...
          from {$g5['g5_shop_cart_table']} a
          where a.od_id = '".get_session('ss_cart_id')."'
            and a.it_id = '{$row['it_id']}'  // ← it_id로 모든 옵션 조회
          order by a.ct_id ";
```

## 🔧 수정 방안

### 수정 1: `shop/cart.php` - AOV 상품 옵션 조회 쿼리 수정

**현재 코드 (113줄)**:
```php
where a.ct_id = '{$row['ct_id']}'
  and a.it_id = '{$row['it_id']}'
```

**수정 후 코드**:
```php
where a.it_id = '{$row['it_id']}'  // ct_id 조건 제거
  and a.od_id = '$s_cart_id'
```

**이유**:
- 같은 상품(it_id)의 모든 옵션을 조회해야 함
- `ct_id` 조건은 불필요 (이미 `it_id`로 필터링됨)

## ⚠️ 주의사항

### 1. 일반 상품과의 일관성

일반 상품의 경우 (57-67줄):
```php
$ct_sql = " select ct_id from {$g5['g5_shop_cart_table']} 
            where od_id = '$s_cart_id' 
              and it_id = '{$row['it_id']}'
              ...
```

일반 상품은 이미 `it_id`로 모든 `ct_id`를 조회하고 있습니다.  
AOV 상품도 동일한 로직을 사용하도록 수정하면 일관성이 유지됩니다.

### 2. 폐쇄몰 분리 확인

수정 시 폐쇄몰 조건이 유지되는지 확인:
- ✅ `AND (a.ct_closed_mall_id IS NULL OR a.ct_closed_mall_id = 0)` 조건 유지

### 3. 성능 고려

- `it_id`로 조회하는 것은 인덱스가 있다면 성능 문제 없음
- `boxcart.skin.php`에서도 동일한 방식으로 조회하고 있어 검증됨

## ⚠️ 추가 확인 필요 사항

### 일반 상품 옵션 표시

**현재 상태**: 일반 상품의 경우 `print_item_options()` 함수를 사용하여 옵션을 표시하고 있습니다.

**확인 필요**:
- `print_item_options()` 함수는 특정 `ct_id`로 옵션을 조회합니다 (lib/shop.lib.php 1505줄)
- 일반 상품도 여러 옵션이 있을 때 모든 옵션이 표시되는지 확인 필요
- 현재는 합계만 계산하고 개별 옵션은 `print_item_options()`로 표시하는 구조

**일반 상품 옵션 표시 로직**:
- `shop/cart.php` 515줄: `print_item_options($item['it_id'], $s_cart_id, $item['ct_id'])`
- 특정 `ct_id`로 옵션 조회 → 같은 상품의 다른 옵션은 표시되지 않을 수 있음

**권장 사항**:
- 일반 상품도 여러 옵션을 표시해야 한다면, AOV 상품과 동일하게 모든 옵션을 조회하여 표시하도록 수정 필요
- 또는 `print_item_options()` 함수를 수정하여 `ct_id` 조건 없이 `it_id`로 모든 옵션 조회

**현재 일반 상품 옵션 표시 방식**:
- `shop/cart.php` 515줄: `print_item_options($item['it_id'], $s_cart_id, $item['ct_id'])`
- 특정 `ct_id`로 옵션 조회 → 같은 상품의 다른 옵션은 표시되지 않을 수 있음
- 합계만 계산하고 개별 옵션은 `print_item_options()`로 표시하는 구조

**일반 상품도 여러 옵션 표시가 필요한 경우**:
- `print_item_options()` 함수 호출 시 `ct_id` 파라미터를 빈 문자열로 전달하거나
- `print_item_options()` 함수 내부에서 `ct_id`가 없으면 모든 옵션을 조회하도록 수정 필요
